### Exercise 1.2 — Add retry with exponential backoff

The current retry uses a fixed 500ms wait. Change it to exponential backoff with jitter.

```yaml
resilience4j:
  retry:
    instances:
      inventory-service:
        maxAttempts: 3
        waitDuration: 200ms
        enableExponentialBackoff: true
        exponentialBackoffMultiplier: 2
        # jitter prevents thundering herd when many services retry simultaneously
        randomizedWaitFactor: 0.5
```

Verify by stopping inventory-service and watching the retry timing in the logs.
The waits should be approximately: 200ms, 400ms (±jitter).

// After saving the order:
MDC.put("orderId", String.valueOf(saved.getId()));
